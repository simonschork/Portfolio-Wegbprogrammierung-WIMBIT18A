Webanwendungen - Web first?

Webanwendungen sind Anwendungsprogramme, die auf einer Client-Server-Architektur basieren. Die Anwendung liegt auf einem Server, dort werden die Rechenoperationen und Datenverarbeitungsprozesse ausgeführt. Die Verteilung, welche Teile der Anwendung auf dem Server und welche im Client ausgeführt werden, kann der Entwickler im möglichen Rahmen selbst bestimmen.
Der Nutzer selbst greift über seinen Client in Form eines Internetbrowsers oder einer ähnlichen Applikation, die als Schnittstelle dient auf die eigentliche Anwendung zu. Dies geschieht über verschiedenste Protokolle. Beim Beispiel des Webbrowsers über das HTTP-Protokoll. Neben HTTP gibt es zum Beispiel FTP, welches das Übertragen von Dateien regelt oder SSH was den Zugriff und die Steuerung verschiedener Server ermöglicht.
Als Abgrenzung, eine Webanwendung muss nicht zwingend über einen Browser und HTTP erreichbar sein. Dieser Irrtum rührt von der Verwechslung des Begriffe Internet und World Wide Web. Webseiten und andere Anwendungen, die über einen Hyperlink im HTTP-Format erreichbar sind, sind im World Wide Web. Handelt es sich um eine Anwendung der oben genannten Beispielprotokolle, wie E-Mail oder Datentransfer via FTP, so ist diese auch im Internet und somit eine Webanwendung, jedoch nicht im World Wide Web.

Ein enormer Vorteil solcher Webanwendungen ist die Plattformunabhängigkeit. Internetbrowser sind auf den meisten Geräten vertreten und bieten somit eine einheitliche Schnittstelle. Dies mindert auch den Entwicklungsaufwand enorm. Änderungen müssen einzig auf dem Webserver vorgenommen werden und sind unabhängig von den jeweiligen Betriebssystemen der Nutzer.
Nachteil hingegen ist die Notwendigkeit einer Verbindung zum Webserver. Diese ist abhängig von Standort und Gerät des Nutzers. Offlinenutzung von Webanwendungen ist somit unmöglich.
